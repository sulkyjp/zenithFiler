# Zenith Filer ユーザマニュアル (v0.9.4)

> **Zenith Filer** は、効率的なファイル操作を追求した Windows 用の **2 画面タブ型ファイルエクスプローラー**です。標準エクスプローラーに近い操作感を維持しつつ、パワーユーザー向けの高度なカスタマイズ機能を提供します。

**このマニュアルの見方**: 左側の目次をクリックすると該当セクションへスムーズにジャンプします。スクロールに応じて、現在読んでいるセクションが目次で自動ハイライトされます。

---

## 1. 🖥️ 画面構成

> **アプリの各パーツと役割を把握する** — タイトルバー、ファイルペイン、ナビペイン、ステータスバーで構成されています。

### 1-0. ⚡ 起動時のスプラッシュ画面

**初回起動時のみ**、WPF のネイティブスプラッシュ画面（静止画）が表示されます。チャコールブラックの背景にアプリロゴと「初回起動の準備をしています…」のメッセージが表示され、データベースやインデックスの初期化が完了すると 0.5 秒かけてフェードアウトし、メインウィンドウへ遷移します。

- OS レベルで即座に表示されるため、起動が始まったことがすぐに分かります。
- **2 回目以降はスプラッシュを完全にスキップ**し、メインウィンドウを即座に表示する「爆速起動」を実現しています（`settings.json` の有無で初回起動を判定）。

### 1-1. 🏷️ タイトルバー

タイトルバーには、アプリ名・バージョンに続けて、**フォーカス中のペイン**のフォルダのフルパスが表示されます（例: `Zenith Filer - C:\Users\...\Desktop`）。パスが空のときや起動直後はアプリ名とバージョンのみです。**Tab** キーでペインを切り替えると、表示パスも連動します。

### 1-2. 📂 ファイルペイン (Aペイン / Bペイン)

> **ファイル操作の中心エリア** — 左右独立したペインで、タブブラウジング・複数フォルダの並行作業が可能です。

- **タブ機能**: 複数のフォルダをタブとして開き、切り替えながら作業できます。
- **Aペイン（左側）/ Bペイン（右側）**: それぞれ独立して動作し、**クリック**または **Tab** キーでフォーカスを切り替えます。フォーカス中のペイン（A/B のみ。ナビペインは対象外）は背景がわずかに濃く、文字色が黒で表示されます。上部メニュー（タブヘッダー・アクション行・アドレスバー）は一覧部分よりわずかに濃い色で表示され、アクティブペインの判別を助けます。

#### ペインヘッダー（2行レイアウト）

各ペインの上部ヘッダーは **2 行構成**です。

**1 行目: アクション行** — 機能ごとに 5 グループ（セパレーター区切り）で配置:

| グループ | ボタン |
| :--- | :--- |
| **ナビゲーション** | 戻る ／ 進む ／ 上へ |
| **ショートカット** | ホーム ／ デスクトップ ／ ダウンロード |
| **作成・編集** | 新規フォルダ ／ フォルダを取り込む |
| **管理** | 更新 ／ エクスプローラーで開く |
| **表示設定** | アダプティブ列 ／ フォルダを先に |

- **ホーム**ボタンはアプリ設定で指定したフォルダに移動します。未設定時は A ペイン→デスクトップ、B ペイン→ダウンロードへ移動します。
- 右端には**検索ボックス**があります。通常タブでは **Enter** で新規タブに検索結果を表示。検索結果タブでは入力と同時に即時検索が実行されます。検索ボックス内右端にはヒット件数がリアルタイム表示されます。
- 検索中はリストの列構成が専用のもの（名前・場所・パス・更新日時・サイズ）に切り替わり、ヘッダークリックで並び替え、ヘッダー境界のドラッグで列幅調整、ダブルクリックで AutoFit できます（手動調整時はウィンドウ幅連動の自動伸縮が無効）。パス列は末尾が優先表示されます。

**2 行目: アドレスバー行（通常時）／フィルターバー（検索時）**

- 通常時はアドレスバー（ブレッドクラム／パス入力）を横幅いっぱいに配置。左端に「パスを編集」ボタン（鉛筆アイコン）、右側に「お気に入りに追加」ボタン（★）と「パスをコピー」ボタン。
- **検索実行時**はフィルターバーに切り替わり、ファイル種類で絞り込めます。

#### 検索履歴ドロップダウン

検索ボックスに**フォーカス**すると、過去の検索キーワード（最大 100 件）がドロップダウン表示されます。

- **虫眼鏡アイコン** = 通常検索の履歴、**雷アイコン** = インデックス検索の履歴
- 同一キーワードでも通常 / インデックスは別履歴として表示されます。
- 項目を選択（クリックまたは **Enter**）すると、記録時の検索モードに自動切替して再検索されます。
- 履歴表示中に **↓** でリストへフォーカス、**↑↓** で項目移動、**Enter** またはクリックで決定。検索ボックスにフォーカス中の **↑** では履歴に移りません。

#### 検索フィルターバー

検索実行時、2 行目のアドレスバーがフィルターバーに置き換わります。

- フォルダ・Excel・Word・PPT・PDF・TXT・EXE・BAT・JSON・画像・その他のチェックボックスで表示種類を絞り込めます。
- 左端の「全選択」「全解除」ボタンで一括操作。
- チェックを外した種類は検索結果に表示されません。
- **フィルター選択状態は `settings.json` に保存され、再起動後も復元されます。**
- 検索バー右端の **×** または **Esc** で検索条件をクリアできます。

#### ブレッドクラムバー

現在のパスを階層ごとにボタン形式で表示します。

- 各階層をクリックすると直接移動できます。
- **ファイルをドラッグしてパンくずの階層にドロップ**すると、その階層へコピーまたは移動できます（左ドラッグ: Ctrl でコピー、なしで移動。右ドラッグ: メニューで選択）。
- 各階層右側の **▼** をクリックするとサブフォルダ一覧がドロップダウン表示され、選択で移動できます（エクスプローラーと同じ動作）。
- パスが表示幅を超えた場合は先頭階層から「**...**」で省略されます。省略ボタンをクリックすると隠れた階層がドロップダウンで表示されます。

#### その他の機能

- **横スクロール**: ファイルリストやブレッドクラムバーでマウスホイールの左右チルトが使えます。
- **パス編集モード**: アドレスバー左端の**鉛筆アイコン**（トグル）をクリック、またはバーの空白部分をクリックすると、パスを直接テキスト入力できるモードに切り替わります。オンにすると右矢印アイコン（ブレッドクラム表示への戻り）に変わります。バー右側の「フォルダを開いたアイコン」はエクスプローラーで開くボタン、「取り込みアイコン」はエクスプローラーのフォルダを取り込むボタンです。
- **オーバーレイ・スクロールバー**: ファイルリストの垂直スクロールバーはコンテンツの上に重なる形式で表示されます。スクロールバーが表示されてもファイル名列の幅が変動せず、常に安定したレイアウトです。
- **お気に入り登録ボタン（★）**: アドレスバー右側（コピーボタンの左）に配置。クリックで「お気に入りに追加」ダイアログが開き、名前と概要説明（任意）を入力して登録できます。登録成功時はナビペインがお気に入りビューへ自動切替し、追加項目が一時的に強調表示されます。ロック中でも登録可能です。

### 1-3. 🧭 ナビペイン (サイドバー)

> **素早いフォルダアクセスの拠点** — お気に入り・ツリー・履歴・インデックス検索設定の 4 モードと、アプリ設定を切り替えて使用します。

画面左側の領域で、左側の**ラジオボタン**でモードを、右側の**歯車アイコン**でアプリ設定を切り替えます。

#### ⭐ お気に入りビュー (Ctrl+Shift+1)

よく使うフォルダやファイルを登録・整理できます。ドラッグ＆ドロップでの登録や、仮想フォルダによる階層化が可能です。

- 各フォルダには**概要説明**を追加でき、似た名前のフォルダを識別しやすくなります（例:「BOX用スクリプト (週次報告用)」）。
- リスト上部の**検索ボックス**にキーワードを入力すると一覧を絞り込み表示します（検索中はツリー階層が解除され、ヒット項目がフラットに表示）。
- **検索モード**はナビペイン右下のボタン（ナビ幅ロックの左）で切替。**名前・概要**モードではフォルダ名/概要でヒット、**フルパス・概要**モードではフルパス/概要でヒットします。
- **Esc** または検索ボックス右端の **×** で検索をクリアできます。

#### 🌳 ツリービュー (Ctrl+Shift+2)

PC 内の全ドライブとフォルダ階層をツリー形式で表示します。

- 各フォルダはフルパスで一意に管理され、重複なくもれなく一覧できます。
- A/B ペインでのフォルダ作成・リネーム・移動・削除はツリーに即座に反映されます。
- このビューに切り替えると、深いパスが省略されないようナビペイン幅がアニメーションで広がり、お気に入りに戻すと滑らかに元の幅に戻ります。
- ツリービュー表示時のみ、右下に**ツリービューロック**（南京錠）ボタンが表示され、ロック中はドラッグ＆ドロップ移動・リネーム（**F2**）・削除（**Delete**）が禁止されます。禁止操作を試みると**ナビペインロック仕様**に従い通知されます。

**ツリービューの操作:**

| 操作 | 動作 |
| :--- | :--- |
| **ワンクリック** | フォルダを選択するだけ。ペインには表示しない |
| **展開矢印（▶/▼）のクリック／ダブルクリック** | ツリーの展開・折りたたみのみ。ペインには表示しない |
| **行本体のダブルクリック** | そのフォルダを**フォーカス中のペイン**に表示（ツリーの展開状態は変わらない） |
| **Ctrl+ダブルクリック** | そのフォルダを**反対側のペイン**に表示（1ペイン時は自動2ペイン化） |
| **Enter** | フォルダを選択した状態で、**フォーカス中のペイン**に表示 |
| **Shift+クリック** | そのフォルダを**フォーカス中のペイン**に**大アイコン（アイコンビュー）**で表示 |
| **Shift+Ctrl+クリック** | そのフォルダを**反対側のペイン**に**大アイコン（アイコンビュー）**で表示（1ペイン時は自動2ペイン化） |
| **右クリック** | 「Aペインに表示」「Bペインに表示」等のメニューで表示先を明示的に指定 |

各フォルダにマウスを載せると、上記操作方法をツールチップで確認できます。

#### 🕐 参照履歴 (Ctrl+Shift+3)

過去にアクセスしたフォルダの履歴を日別に表示します。

- 日付グループは「2026/02/04(水)」のように曜日付き。
- 上部の検索ボックスにキーワードを入力すると、履歴全体から検索し一覧表示（参照日付の降順）に切り替わります（日付グループ分けは解除）。
- このビューに切り替えるとナビペイン幅がアニメーションで広がり、お気に入りに戻すと元の幅に戻ります。
- お気に入りビュー時のスプリッター幅は設定に保存されます。ビュー切り替え時の幅変更は約 0.4 秒のアニメーション。
- 参照履歴をすべて削除するゴミ箱（履歴削除）ボタンは、過去履歴ビューの**右下**にあります。

#### ⚡ インデックス検索設定 (Ctrl+Shift+4)

インデックス検索の対象フォルダを登録・管理します。

- このビューに切り替えるとナビペイン幅がアニメーションで広がります。
- 「フォルダを追加」でツリーから選択、「現在のフォルダを追加」で A/B ペイン表示中のフォルダを追加。
- ファイル一覧やコンテキストメニューから「インデックス検索でこのフォルダを検索」を選ぶと、未登録の場合は自動登録され、このビューへ切り替わって追加項目が強調表示されます。
- 一覧で項目を選択して「削除」ボタンまたは右クリックメニューから削除。
- **右クリックメニュー**: 「インデックスを再作成」（削除→再作成）、「インデックスを差分更新」（新規・更新分のみ反映）、「インデックス登録確認」（A ペインの新規タブに一覧表示、タブ名「結果確認：（フォルダ名）」）。
- 登録一覧は設定に保存され、次回起動時に復元されます。
- **インデックスの状態**ブロック: 現在の状態（完了／差分更新中／一時停止中）と総件数を表示。「未インデックスを作成」「一時停止」「再開」で制御可能（検索は一時停止中も利用可能）。
- Box フォルダはスキャン前に暖気運転（スタブ生成促し）を自動実行。
- メニュー（⋯）から「フル再構築を実行…」「インデックスの設定を開く」にアクセス。

#### ⚙️ アプリ設定 (Ctrl+Shift+O)

ナビペイン右側の歯車アイコン（マニュアルの左）をクリックするか、**Ctrl+Shift+O** で切り替え。このビューに切り替えるとナビペイン幅がアニメーションで広がります。

**1. 基本設定 (Home & General)**

| 設定項目 | 効果 |
| :--- | :--- |
| **コンテキストメニュー: アプリ独自メニュー（推奨）** | 開く・コピー・削除・名前の変更等を即表示。**Shift+右クリック**またはメニュー内の項目でエクスプローラ互換メニューに遷移可能 |
| **コンテキストメニュー: エクスプローラ互換メニュー** | 7-Zip 等のシェル拡張含む Windows 標準メニューを表示 |
| **A/B ペインのホームアドレス** | 「表示中のフォルダを設定」で現在のフォルダを、「フォルダを選択」でダイアログから選択して設定 |

**2. 検索 (Search UI)**

| 設定項目 | 効果 |
| :--- | :--- |
| **検索の挙動: 同一ペイン・新規タブ** | **Enter** で新規タブに結果を表示（デフォルト） |
| **検索の挙動: 同一ペイン・現在タブで即時検索** | 入力即検索（**Enter** 不要）。文字を消すと一覧に戻る |
| **検索の挙動: 反対ペイン・新規タブ** | **Enter** で反対ペインに結果を表示。1ペイン時は自動2ペイン化 |
| **パスクリック: 同一タブで表示** | 検索結果ビュー→通常一覧に切り替え |
| **パスクリック: 同一ペイン・新規タブ** | 同ペインに新規タブで開く（デフォルト） |
| **パスクリック: 反対ペイン・新規タブ** | 反対ペインに新規タブで開く。1ペイン時は自動2ペイン化 |
| **検索時に自動的に1画面モードに切り替える** | 検索時に1ペインモードに切り替え、結果を広く表示。Bペインで検索した場合はAペインに移動 |

**3. インデックス (Indexing & Freshness)**

| 設定項目 | 効果 |
| :--- | :--- |
| **変更を自動で取り込む** | FileSystemWatcher で変更を即時検出し差分更新 |
| **一定間隔でまとめて更新する** | 1h / 2h / 4h / 1day ごとに全体再スキャン |
| **必要なときだけ更新する** | 手動実行時のみ更新。起動時・フォルダ追加時の自動インデックスなし |
| **省エネモードでインデックスを更新する** | CPU・ディスク使用を抑え他アプリへの影響を軽減 |
| **ネットワークドライブは軽めに処理する** | UNC / マップドドライブをゆっくり処理し帯域・サーバ負荷を節約 |
| **フル再インデックス最短間隔** | 6h / 12h / 24h 未満の再実行はスキップ |
| **できるだけ最新の状態を保つ（鮮度優先）** | 検索時にバックグラウンドで差分更新・未インデックス作成を並行実行 |
| **インデックスが古い可能性があるときにメッセージを表示する** | 未インデックスフォルダがある場合に検索結果に警告バーを表示。右端の **×** で閉じ可能 |

設定は次回起動時に復元されます。

#### その他のナビペイン設定

- **お気に入り検索モード**: お気に入りビュー時、ナビペイン右下の**ナビ幅ロックの左**に検索モード切替ボタン（Type／Route アイコン）があります。オフ時は「名前・概要」、オン時は「フルパス・概要」でヒットします。状態は次回起動時に復元されます。
- **ナビ幅ロック**: お気に入りビュー時のみ、ナビペイン右下にパネルアイコン（開いたパネル／閉じたパネル）のトグルが表示されます。オンにするとナビペイン幅をスプリッターで変更できなくなります。ツリー・履歴への切替時の自動拡張はロック時も有効。状態は次回起動時に復元されます。
- **ナビペインロック仕様（共通）**: お気に入りロック・ツリービューロックはいずれも同じ仕様に従います。ロック中に禁止操作を試みると、**ポップアップは出さず**、ステータスバーに「ロック中のため〜できませんでした。」と表示され、南京錠アイコン付近で矢印＋点滅の解除促しアニメーション（約 3 秒間）が表示されます。アニメーション中でも南京錠をクリックすれば即座にロック解除されます。
- **ロック機能はビューごとに独立**: **お気に入りロック**はお気に入りビュー内での仮想フォルダ追加・削除・並び替えのみを制限します（登録・名前変更・概要変更はロック時も可能）。**ツリービューロック**はツリービュー表示時のみ有効で、フォルダの移動・リネーム・削除を禁止します。お気に入りをロックしたままツリーで整理したり、その逆も可能です。

### 1-4. 📖 ヘルプ・ドキュメントビューア

> **マニュアルと更新履歴を 1 つのウィンドウで閲覧** — タイトルバー右側の「マニュアル」アイコンから開けます。

タイトルバー右側の「マニュアル」アイコンから開くか、配布フォルダの **ZenithDocViewer.exe** を直接起動できます（本体不要で軽量起動、事前のドキュメント確認に便利）。

- **総合ドキュメントビューア**: 「ユーザーマニュアル」と「更新履歴（CHANGELOG）」を 1 つのウィンドウ内で切り替えて閲覧できます。**スクロール位置はそれぞれ独立保持**され、タブを切り替えても前の続きから読めます。
- **上部ヘッダー**:
  - 左側にアプリ名とドキュメント用アイコン（BookOpen）。
  - 中央に「マニュアル」「更新履歴」のセグメントボタン。選択中のモードはアクセントカラー（青）で強調。
  - 右側にキーワード検索ボックス。一番右端に「×」ボタン（**Esc** でも閉じ可能）。
- **左ペイン（目次）**:
  - マニュアルモードでは `H1〜H3` の見出しを自動解析し階層付きリストで表示。
  - 更新履歴モードではバージョン見出し（例: `## [0.3.66] - 2026-02-06`）から `v0.3.66` ラベルを生成して一覧表示（新しい順）。見出し末尾に「 : 概要」があれば目次にも反映。
  - 項目クリックで右ペインの該当セクションへスムーズスクロール。本文スクロールに応じて目次が自動ハイライト。
  - 幅はスプリッターで調整可。デフォルトで大きめに表示され、長い見出しも途切れにくいレイアウト。
- **右ペイン（本文）**:
  - Markdown ベースのレンダリングエンジンを共通利用。マニュアルと更新履歴の両方で、**青のアクセントカラー**・テーブルパディング・`kbd` ライクなショートカット表記・行間やドット（Bullet）デザインが統一されています。
- **操作とショートカット**:
  - **Esc** またはヘッダー右側の「×」でウィンドウを閉じます。
  - マニュアル／更新履歴の切り替えはマウスクリック（メインウィンドウのショートカットとは干渉しません）。

### 1-5. 📊 ステータスバー

> **操作の結果確認と画面制御** — 通知メッセージ、ウィンドウ配置、進捗表示などを集約しています。

画面下部の領域で、以下の情報を表示し設定を変更できます。

- **通知メッセージ（中央）**: コピーや移動等の操作完了時にメッセージがふわっと表示され、3 秒後に消えます。
- **項目情報**: 現在選択中のファイル数や合計ファイルサイズが表示されます。
- **画面分割切り替え**: 1 画面（A ペインのみ）と 2 画面（A・B 両方）をボタンで切り替え。初期状態は 2 画面。
- **ナビペイン表示切替**: サイドバーの表示・非表示を切り替え。非表示にするとファイルペインがウィンドウ幅いっぱいに広がります。
- **常に最前面に表示**: ピンアイコンをクリックすると他のウィンドウより常に手前に表示。リネーム入力やマニュアルなどのポップアップは最前面固定時もメインウィンドウより前面に表示されます。
- **ツールチップ**: ボタンやアイコンにマウスを乗せると操作内容を示すツールチップが表示されます。ロック・固定など状態が変わるボタンは文言が動的に切り替わります。

#### ウィンドウ配置制御

ステータスバー左端のボタンで、ウィンドウの配置を素早く変更できます。

| ボタン | 動作 |
| :--- | :--- |
| **左半分に配置** | 画面の左半分にスナップ |
| **最大化** | 画面全体に広げる（再度押すと元に戻る） |
| **下半分に配置** | 画面の下半分にスナップ |
| **右半分に配置** | 画面の右半分にスナップ |
| **左下に配置** | 画面の左下 1/4 にスナップ |
| **右下に配置** | 画面の右下 1/4 にスナップ |

- **最大化からのドラッグ解除**: 最大化状態でタイトルバーをドラッグすると、直前の通常サイズ（無効時はディスプレイ作業領域の約 75%幅・16:10 比率）にリサイズされ、カーソルがタイトルバー中央付近に来るよう位置補正されます。
- **マルチディスプレイ自動レスキュー**: 前回と異なるディスプレイ構成で起動し、ウィンドウが画面外にある場合はメインディスプレイの中央付近に自動再配置します。

#### 進捗表示の種類

| 種類 | 表示場所 | 用途 |
| :--- | :--- | :--- |
| **スピナー** | ステータスバー右側（ログフォルダボタンの左） | ファイル検索・読み込み・CSV 出力スキャン等、所要時間不定の非同期処理 |
| **スキャンバー** | ステータスバー最上部（横伸びライン） | 大規模スキャン処理。状況メッセージ付き。スピナーと同時表示あり |
| **グロウバー** | ステータスバー最上部（発光プログレスバー） | コピー・移動中。バイト数・件数ベースの進捗をリアルタイム反映。キャンセル時は「吸い込まれ」逆行アニメーション |

- **インデックス作成中アイコン**: ステータスバー右側に「インデックス作成中: 〇〇件 (フォルダ名)」と回転アニメーション。中断後の再開時は前回からのカウント続行。クリックで中止可能。完了時は「〇〇件をインデックスに追加しました」と表示。

#### 表示モード切り替え

各ペインのツールバー（「フォルダを取り込む」の右、「列を自動補正」の左）に以下のボタンが並びます。

- **一覧ビュー**（Rows3 アイコン）: 詳細一覧（名前・更新日時・種類・サイズ等の列表示）に切り替え。
- **小・中・大アイコン**: アイコン表示サイズを切り替え。選択中のモードはボタンのハイライトで判別。

**アイコンモードの特徴:**
- エクスプローラーに近いレイアウト。大・中・小で明確なサイズ差。ペイン端との余白を確保し、アイテムは詰めて配置、余白は右端に集約。
- 文字は通常サイズ。長い名前は「...」で省略。
- **フォルダ・文書など非画像**: カード高さ固定、アイコンとファイル名の開始位置を統一。ファイル名は最大 4 行、以降は省略。
- **画像ファイル**（jpg, png, gif, bmp, webp 等）: サムネイルがカード全面に表示。通常時はファイル名非表示。ホバー時に下部オーバーレイで名前・詳細がフェードイン、枠とシャドウで強調。フルファイル名はオーバーレイ部分にホバーして約 2 秒後にツールチップで表示。
- **フォルダや画像以外**（文書など）: 四角枠なしでアイコンを上部に、ファイル名をその下の固定位置に常時表示（WindowsAPICodePack-Shell ライブラリ使用）。
- ファイル数が少なくても上から詰めて表示。選択ハイライトとクリック範囲はカード部分のみ（カード外の余白クリックでは選択されない）。
- **アイコンビュー中**は **F5** キーおよび矢印・**Enter**・**Back** 等のキー操作は無効。更新はツールバーの「更新」ボタン（マウスクリック）で実行。詳細表示に切り替えると **F5** 含め従来通り利用可能。
- **表示モードは次回起動時にも引き継がれ**、A/B ペインそれぞれ独立で保存。新規タブはそのペインの表示モードが自動適用。

## 2. 📁 基本的なファイル操作

> **日常のファイル操作を効率化** — ナビゲーション、タブ操作、ファイル操作、D&D をカバーします。

### 2-1. 🗺️ ナビゲーション

| 操作 | 方法 |
| :--- | :--- |
| **フォルダを開く** | ダブルクリック、または **Enter** |
| **戻る** | ツールバー「←」、または **Alt+←**（戻った先では直前のフォルダにフォーカス） |
| **進む** | ツールバー「→」、または **Alt+→** |
| **上の階層へ** | ツールバー「↑」、または **Backspace**（戻った先では直前のフォルダにフォーカス） |
| **ホームへ移動** | ツールバーのホームアイコン（未設定時: A→デスクトップ、B→ダウンロード） |
| **デスクトップへ移動** | ツールバーのモニターアイコン |
| **ダウンロードへ移動** | ツールバーのダウンロードアイコン |
| **お気に入りに追加** | アクション行の「★」ボタン |
| **エクスプローラーで表示** | ツールバーのフォルダを開いたアイコン |
| **エクスプローラーのフォルダを取り込む** | ツールバーの取り込みアイコン。1 件→新規タブで表示しエクスプローラーを閉じる。2 件以上→選択ダイアログで対象を選び、各タブで表示して閉じる |
| **パスをコピー** | アドレスバー右端のコピーアイコン |
| **更新** | **F5**、またはツールバーの更新ボタン。アイコンビュー時は**ボタンのみ有効** |

### 2-2. 🗂️ タブ操作

| 操作 | 方法 |
| :--- | :--- |
| **新しいタブ** | タブバー右の「＋」ボタン、**Ctrl+T**、またはタブバー空白部分のダブルクリック |
| **タブを閉じる** | タブの「×」ボタン、または **Ctrl+W** |
| **タブ切り替え** | タブをクリック、または **Ctrl+Tab**。フェードイン＋インジケータのスライドアニメーション付き |
| **タブの移動** | タブをドラッグして左右に移動し並び順を変更 |
| **D&D で開く** | フォルダをタブバー空白部分にドロップ→新規タブで開く |

- **タブ一覧の表示**: タブバー右端（＋の右）のタブ一覧ボタン（List アイコン）をクリック。プルダウンで全タブを一覧表示し、クリックでタブ切替、各タブ右側の「×」で削除。現在タブは青ハイライト、ロックタブは削除ボタン非表示。一覧外クリックまたはタブ選択で自動的に閉じます。
- **タブのロック**: タブ右クリック→「ロック」で閉じ不可に。南京錠アイコンが表示され「×」が非表示。再度右クリック→「ロック解除」で解除。状態は次回起動時に引き継がれます。
- **全タブをお気に入りに追加**: タブ右クリック→「このペインのタブを全てお気に入りに追加」で、全タブのパスを新規仮想フォルダにまとめて登録。名前と概要（任意）をダイアログで入力。存在しないパスや登録済みパスはスキップ。
- **タブを反対ペインに移動**: タブ右クリック→「反対ペインに移動」。1ペイン時は自動 2 ペイン化。

### 2-3. ✂️ 標準的なファイル操作

> **右クリックメニューまたはショートカットキーで操作。** デフォルトでは**アプリ独自の軽量メニュー**が即表示されます。

- **コピー・切り取り・貼り付け**: **Ctrl+C** / **Ctrl+X** / **Ctrl+V**（Windows 標準と同様）。操作後もフォーカスはファイルリストに残り、コピー・切り取り時は操作項目に、ペースト後はペーストしたファイルにフォーカス。自動スクロールで即座に視認でき、フォルダ監視の自動更新後もフォーカスが維持されます。
- **新規フォルダ**: **Ctrl+Shift+N** で名前の変更ウィンドウが即座に表示。入力して OK で作成（バックグラウンド実行のためネットワークドライブ等でもダイアログはブロックされにくい）。作成後はそのフォルダにフォーカス。エクスプローラのコンテキストメニューから作成した場合は従来通り。テキストボックス右下の**「本日の日付」ボタン**で `YYYYMMDD` 形式の日付を自動入力。
- **名前の変更**: **F2** またはメニュー。ファイルは名前部分のみ表示され拡張子は変更不可。フォルダは名前全体を編集可。ダイアログはキャッシュされ瞬時に表示。**「本日の日付」ボタン**で `YYYYMMDD` 形式に変更可能。
- **削除**: **Delete** キー。確認ダイアログ後、ごみ箱へ送られます。
- **ショートカットの作成**: 選択項目のショートカットを現在のフォルダに作成。
- **プロパティ表示**: Windows 標準のプロパティ画面を表示。
- **エクスプローラーで開く**: 現在のフォルダをエクスプローラーで開きます。

### 2-4. 📋 独自コンテキストメニュー

> **アプリ独自メニュー（軽量・高速）とエクスプローラ互換メニュー（シェル拡張対応）の 2 種類** — デフォルトはアプリ独自メニューです。

#### 2-4-1. メニューの種類と切り替え

| メニュー | 特徴 |
| :--- | :--- |
| **アプリ独自メニュー（推奨）** | よく使う操作を即座に表示する軽量メニュー。Windows 11 風のモダンスタイル（角丸・余白・透明感）で、右クリック位置に待機時間なく即表示 |
| **エクスプローラ互換メニュー** | Windows 標準の右クリックメニュー。7-Zip 等のシェル拡張やアプリ固有項目も利用可能 |

**切り替え方法:**
- **アプリ設定で変更**: ナビペイン→「アプリ設定 (**Ctrl+Shift+O**)」→「1. 基本設定」→「コンテキストメニュー」。設定は次回起動時に引き継がれます。
- **一時的に切り替え**: アプリ独自メニューがデフォルトの場合、**Shift+右クリック**でエクスプローラ互換メニューを表示。また独自メニュー内の「エクスプローラのコンテキストメニューを表示する」でも切替可能。
- **Box フォルダでの共有リンク自動整形**: Box Drive 内の項目で **Shift+右クリック**すると、最大 20 秒間クリップボードを監視。「共有」→「共有リンクをコピー」を選ぶと、Box URL を検知した時点で「1 行目: Box 相対パス、2 行目: 共有 URL」の 2 行形式に自動置換。成功時はグリーンフラッシュ＋ステータスバー通知。

#### 2-4-2. アプリ独自メニューの項目

選択した項目の種類や数に応じて適切な操作項目を自動表示します。区切り線で **開く・表示** → **クリップボード** → **名前・パスのコピー** → **編集・変更** → **アプリ固有** → **その他・システム** の 6 グループに分類。

**ファイル・フォルダを選択した場合:**

- **開く**: ファイルは既定アプリで開く。フォルダはそのまま開く。
- **新しいタブで開く**（フォルダのみ）: 新規タブでフォルダを開く。
- **反対ペインで開く**（フォルダのみ）: 反対ペインで開く（1ペイン時は自動2ペイン化）。
- **テキストエディタで開く**（テキスト系ファイルのみ）: `.txt`, `.md`, `.json`, `.html` 等を `.txt` の既定アプリで開く。複数選択時は全てテキスト系の場合に表示。
- **ペインターで開く**（画像系ファイルのみ）: `.png`, `.jpg`, `.bmp` 等を OS 設定の画像編集アプリで開く。複数選択時は全て画像系の場合に表示。
- **コピー** / **切り取り** / **貼り付け**: **Ctrl+C** / **Ctrl+X** / **Ctrl+V** と同じ。
- **ファイル名をコピー**: 名前をクリップボードにコピー。複数選択時は改行区切り。
- **フルパスをコピー**: フルパスをクリップボードにコピー。複数選択時は改行区切り。
- **連携用BOXパスをコピー**（Box 領域のみ）: `Box\～` 形式のパスをコピー。複数選択時は改行区切り。実 URL 付き 2 行形式が必要な場合は **Shift+右クリック** → 共有リンクコピーで自動整形。
- **削除**: **Delete** と同じ。確認ダイアログあり。
- **名前の変更**: **F2** と同じ。単一選択時のみ表示。
- **ショートカットの作成**: 選択項目へのショートカットを現在のフォルダに作成。
- **このフォルダをA/Bペインのホームに設定**（フォルダのみ）: 選択フォルダをホームに設定。
- **お気に入りに追加**（フォルダ・ファイル）: お気に入りに登録。既に登録済みの場合は「お気に入りから解除」を表示。
- **PDFに変換して同フォルダへ保存**（単一・変換対応形式のみ）: 画像（`.jpg`, `.png`, `.bmp`, `.gif`）や Office 文書（`.docx`, `.xlsx`, `.pptx` 等）を PDF に変換して同一フォルダへ保存。Office 文書の変換には Microsoft Office が必要。変換中はグロウバーで進捗表示。
- **PDFに変換して結合・同フォルダへ保存**（複数・全て変換対応形式の場合）: 複数ファイルを PDF 変換後、1 つに結合して `Combined_yyyyMMddHHmmss.pdf` として保存。
- **インデックス検索でこのフォルダを検索**（フォルダのみ）: 選択フォルダをインデックス検索。
- **このフォルダ以下の一覧をCSV出力**（フォルダのみ）: サブフォルダ含む全ファイルをスキャンし CSV 生成（背景右クリック版と同仕様）。
- **プロパティ**: Windows 標準のプロパティ画面。単一選択時のみ。
- **エクスプローラのコンテキストメニューを表示する**: エクスプローラ互換メニューに切替。

**背景（空きスペース）を右クリックした場合:**

- **新しいフォルダ**: **Ctrl+Shift+N** と同じ。作成直後にリネームダイアログ表示。キャンセル時は自動削除。
- **新しいテキストファイル**: 空の `.txt` を作成。作成直後にリネームダイアログ表示。キャンセル時は自動削除。
- **貼り付け**: クリップボードの内容を貼り付け。
- **フルパスをコピー**: 現在のフォルダのフルパスをコピー。
- **フォルダ内の一覧をコピー（名前）**: 表示中の項目名を改行区切りでコピー（フィルタ・ソート反映。検索結果タブ時は検索結果の名前）。
- **フォルダ内の一覧をコピー（フルパス）**: 表示中の項目のフルパスを改行区切りでコピー。
- **連携用BOXパスをコピー**（Box 領域のみ）: 現在フォルダの `Box\～` パスをコピー。
- **インデックス検索でこのフォルダを検索**: 現在フォルダをインデックス検索。
- **このフォルダ以下の一覧をCSV出力**: サブフォルダ含む全ファイルをスキャンし UTF-8 BOM 付き CSV（列: 項番・ファイル名・フルパス・タイムスタンプ・サイズ）を生成。保存先ダイアログの初期フォルダは対象フォルダ。スキャン中はスキャンバー＋スピナー同時表示。拡張子なしファイルは対象外（件数上限なし）。
- **プロパティ**: 現在フォルダのプロパティ画面。
- **エクスプローラのコンテキストメニューを表示する**: エクスプローラ互換メニューに切替。

#### 2-4-3. メニューの表示位置とスタイル

- **表示位置**: 右クリック位置を起点に表示。画面端にはみ出す場合は自動補正。
- **表示速度**: 待機時間なく即座に表示。
- **デザイン**: Windows 11 風モダンスタイル（角丸・余白・透明感）。各項目にアイコン、区切り線でグループ分け。

#### 2-4-4. エクスプローラ互換メニューについて

エクスプローラ互換メニューは Windows 標準の右クリックメニューをそのまま表示します。以下のような場合に便利です:

- 7-Zip などのシェル拡張を利用したい場合
- 特定アプリが追加したコンテキストメニュー項目を使用したい場合
- Windows 標準の動作に慣れている場合

リネームや削除などの操作もアプリ側で適切に処理されるため、操作感はアプリ独自メニューと同様です。

### 2-5. 🖱️ ドラッグ＆ドロップ

> **ペイン間・外部アプリとのファイル移動やコピー** — Windows 標準準拠のドラッグ操作に加え、独自の便利機能を搭載しています。

- **複数選択とドラッグ**: **Ctrl** / **Shift** で複数選択し、ドラッグすると選択中の全項目がコピー／移動の対象。
- ペイン間、または外部アプリケーションとの間でドラッグ＆ドロップでコピー・移動が可能です。
- **ブラウザからの URL ドロップ**: アドレスバーやリンクをファイルリスト / ブレッドクラムバーにドロップ→ `.url` ファイルを作成して保存。
- **ドラッグ中のガイド表示**: ドラッグ開始時にカーソル横に「n件の項目をコピー/移動先にドロップしてください」と表示。

**操作の種類**（Windows 標準準拠）:

| 操作 | 動作 |
| :--- | :--- |
| **Ctrl なしでドロップ** | 移動 |
| **Ctrl を押しながらドロップ** | コピー |

- **ナビゲーション用ドロップ**: お気に入り・参照履歴・ツリーの項目をファイルリストにドロップ→そのフォルダへ遷移。検索結果表示中でもドロップ可能（検索が解除され通常表示に戻ります）。
- **一覧からナビペインへのドロップ**: **お気に入りビュー**へ→お気に入りに登録（ロック中でも可能）。**ツリービュー**へ→ドロップ先フォルダへ移動（ツリービューロック ON 時は無効、ステータスバーに通知）。
- **一覧からパンくずへのドロップ**: 任意の階層にドロップでコピー/移動。左ドラッグ: Ctrl なし→移動、Ctrl 押下→コピー。右ドラッグ: メニューで選択。
- ドラッグは約 **10px 以上**動かしたときのみ開始（誤操作防止）。
- Windows 標準の上書き確認やプログレスバーも安全に動作します。

**右クリックドラッグ＆ドロップ（右ドラッグ）:**

| メニュー項目 | 動作 |
| :--- | :--- |
| **ここにコピー** | ドロップ先にコピー |
| **ここに移動** | ドロップ先に移動 |
| **ここにショートカットを作成** | ドロップ先にショートカットを作成 |
| **キャンセル** | 何もしない |

同一ペイン内・ペイン間のどちらでも利用可能。ドロップ先がフォルダ行→そのフォルダ内、それ以外→現在のフォルダが対象。

## 3. 🚀 高度な機能

> **お気に入り管理、表示オプション、検索機能** — Zenith Filer の真価を発揮するパワーユーザー向け機能です。

### 3-1. ⭐ お気に入り管理

> **よく使うフォルダを階層化して素早くアクセス** — 仮想フォルダ、概要説明、ロック機能で整理を強力にサポートします。

- **保存先**: `settings.json` に保存。バックアップすれば別 PC や新バージョンへの移行時に引き継げます。
- **自動バックアップ**: 終了時に `backups/settings_yyyyMMdd_HHmmss.json` へ自動コピー。「4. 設定ファイルリカバリ」から任意時点に復元可能。

#### 追加方法

| 方法 | 動作 |
| :--- | :--- |
| **アドレスバーの★ボタン** | ダイアログで名前と概要（任意）を入力して登録 |
| **ファイル/フォルダをお気に入りビューへ D&D** | 名前・概要ダイアログなしでフォルダ名/ファイル名のまま登録（ロック中でも可能） |
| **右クリック→「お気に入りに追加」** | ファイル・フォルダ選択時に表示 |

| **お気に入り項目を右クリック→「新しいフォルダ」** | その項目と同階層の直下に仮想フォルダを挿入 |
| **お気に入り下部のダウンロードアイコン** | エクスプローラのクイックアクセスを一括インポート（「インポート」フォルダに追加。登録済みはスキップ） |

※ **ツリービュー**へのドロップはお気に入り登録ではなく、ドロップ先フォルダへのファイル移動になります（ツリービューロック ON 時は全て無効）。

#### 整理と操作

- **整理**: 仮想フォルダで階層化。名前は「新しいフォルダ」「新しいフォルダ (2)」…と自動採番。
- **概要説明**: 右クリック→「概要を編集」で追加・変更・削除。ToolTip でパスと概要の全文を確認可能。
- **カテゴリ見出し**: 第 1 階層の仮想フォルダはナビペイン全幅の「スリムな帯」として描画。濃いグレー背景にアクセントライン＋白セミボールド名＋明るいグレー概要。Windows 11 のモダン UI に調和した外観。
- **ファイルの環境アイコン**: お気に入りのファイルはアイコン左側に環境（ローカル／ネットワーク／Box／SPO）が表示されます。
- **パス未検出時の自動確認**: リネーム・削除でパスが見つからない場合、「お気に入りから削除しますか？」と確認。「はい」でロック中でも即座に削除。
- **フォルダとファイルの操作**: フォルダをダブルクリック→ペインに表示。ファイルをダブルクリック→関連アプリで開く。仮想フォルダはダブルクリックで展開/折りたたみ。「A/Bペインで開く」でファイル選択→親フォルダを表示。**ホイールクリック（中クリック）**→フォルダはそのフォルダを、ファイルは親フォルダを、アクティブペインの新規タブで開く（ツリー・検索結果どちらでも利用可能）。

#### ロック機能

- **ナビ幅ロック**: お気に入りビュー時のみ、右下の**左側パネルアイコン**でスプリッター幅変更をロック。ツリー・履歴の自動拡張はロック時も有効。状態は保存・復元されます。
- **お気に入りロック**: お気に入りビュー時のみ、右下の**南京錠アイコン**で仮想フォルダの追加・削除・並び替えを禁止。登録・名前変更・概要変更はロック中も可能。ロック中の禁止操作は**ナビペインロック仕様**に従い通知。
- **ツリービューロック**: ツリービュー時のみ、右下の**南京錠アイコン**でフォルダの D&D 移動・リネーム・削除を禁止。**ファイル一覧からツリーへの D&D** も含め全変更操作を禁止。ロック解除時はツリー上でフォルダを別のフォルダの上（子として）や前後にドロップで実フォルダ移動。禁止操作は**ナビペインロック仕様**に従い通知。状態は次回起動時に復元。
- **削除確認**: サイドバー右下のメッセージアイコンで、削除時の確認メッセージ表示を切り替え。

### 3-2. 👁️ 表示オプション

- **アダプティブ列**: ウィンドウ幅を狭めると「サイズ」→「種類」→「更新日時」→「場所」の順に列が自動非表示になり、最終的にファイル名だけが残る粘り強いレイアウト（検索結果時は「サイズ」→「更新日時」→「場所」→「名前」の順にパス列が最後まで表示）。
- **フォルダを先に表示**: フォルダをファイルより常に上に表示するか、名前順で混在させるかを切り替え。

### 3-3. 🔍 検索機能

> **2 つの検索モード** — 「通常検索（フォルダ内）」と「インデックス検索（横串）」を使い分けて高速にファイルを発見できます。

#### 通常検索（フォルダ内検索）

- **目的**: 現在のフォルダ（およびサブフォルダ）から目的の項目を素早く発見。
- **動作**: 検索ボックスにキーワードを入力し **Enter** で新規タブに検索開始。検索結果タブでは入力と同時に即時検索（Enter 不要）。**Lucene.Net** ベースの検索エンジンで高速応答。
- **検索結果**: ファイルとフォルダの両方がヒットし、更新日時が新しい順に表示。拡張子のないファイルは除外。列ヘッダークリックで並び替え可能。
- パスクリック時の表示先はアプリ設定で選択可能（同一タブ／同一ペイン新規タブ／反対ペイン新規タブ）。

#### インデックス検索モード（横串検索）

- **目的**: カレントフォルダに関わらず、**登録した複数のフォルダ群全体を横断検索**。
- **検索対象**: 「インデックス検索設定」(**Ctrl+Shift+4**) で登録した全パス。ファイルとフォルダの両方がインデックスされます。各フォルダの状態・件数が一覧表示されます。
- **操作**: 検索バー左端の虫眼鏡アイコンをクリック、または **Ctrl+Shift+F** でインデックス検索モード（⚡アイコン・背景色変化）に入ります。
- **検索結果**: ファイルとフォルダの両方がヒットし、更新日時が新しい順に表示。列ヘッダークリックで並び替え可能。

#### インデックスの技術仕様

- **保存場所**: アプリ実行フォルダ内の `index` フォルダ。
- **日本語検索**: 複数語に分割されても全語を含むファイルをヒット。`name_raw` フィールドによるワイルドカード検索で部分一致も確実。
- **ファイル構成**: `segments_N`, `_N.cfs`, `write.lock`, `zenith.db`（履歴DB）等。
- **自動更新**: ファイル操作に応じてバックグラウンドで自動更新。
- **フィルタリング**: 拡張子のないファイルはインデックスに登録されず、検索結果にも非表示（Excel・PDF・画像等のユーザーファイルを素早く発見可能に）。
- **パフォーマンス**: 複数フォルダは 1 件ずつ順次実行。一定件数ごとにスロットリング。`.git` / `node_modules` は自動スキップ。「省エネモード」「ネットワークドライブは軽めに処理する」で負荷をさらに軽減。
- **作成中の検索**: 100 件ごとに検索用インデックスへ反映されるため、作成済み分は検索可能。ステータスバーでアニメーション表示、クリックで中止。
- **件数確認**: 完了時にステータスバー通知。インデックス検索設定ビューで総件数を閲覧可能。
- **進捗表示**: 「インデックス作成中: 〇〇件 (フォルダ名)」が 100 件ごとに更新。
- **フォルダ別の状態・件数**: 「作成中」「未作成」または「〇〇件」表示。完了するたびに即座に更新。
- **フォルダ別の手動更新**: 右クリック→「インデックスを再作成」（削除→再作成で削除済みファイルも反映）、「インデックスを差分更新」（新規・更新分のみ。高速だが削除済みファイルは残る）。
- **検索履歴**: 通常（虫眼鏡）/ インデックス（雷）は別々に保持（最大 100 件）。履歴から選択すると記録時のモードに自動切替して再検索。

#### 3-3-1. 検索の挙動（アプリ設定で変更可能）

アプリ設定（**Ctrl+Shift+O**）の「検索の挙動」で表示先と挙動を選択できます。**デフォルトは「同一ペイン・新規タブ」**。

| オプション | 動作 | 向いている用途 |
| :--- | :--- | :--- |
| **同一ペイン・新規タブ** | **Enter** で検索実行。同じペインに検索結果タブを新規作成。検索結果タブでは入力即検索（Enter 不要）。× で検索タブを閉じて元の一覧タブに戻る | 検索がペイン内で完結し予測しやすい。**初心者にもおすすめ** |
| **同一ペイン・現在タブで即時検索** | 入力即検索（Enter 不要）。現在タブで結果表示。文字を消すか × で通常表示に戻る | 切り替えが素早く、タブを増やさない |
| **反対ペイン・新規タブ** | **Enter** で反対ペインに新規タブで表示。1ペイン時は自動2ペイン化。× で検索タブを閉じて元のタブに戻る | 検索結果と元の一覧を左右で比較できる。**上級者向け** |

#### 3-3-1-1. 検索時の表示設定（アプリ設定で変更可能）

#### 3-3-2. 検索結果のパスクリック時の挙動（アプリ設定で変更可能）

アプリ設定（**Ctrl+Shift+O**）の「検索結果のパスクリック時の挙動」で表示先を選択できます。**デフォルトは「同一ペイン・新規タブ」**。

| :--- | :--- | :--- |
| **同一タブで表示** | 検索結果→通常一覧に切り替え | タブを増やさず素早くフォルダへ移動 |
| **同一ペイン・新規タブ** | 同ペインに新規タブで開く（従来） | 検索結果タブを残したまま別タブで開く。**デフォルト** |

## 4. 🔎 インデックス検索のしくみとチューニング

> **インデックス検索の概念、設定、技術的なしくみをまとめて解説** — 操作や設定の選び方が分かる初学者向けの内容と、実装に興味がある上級者向けの技術説明を含みます。

### 4-1. 💡 インデックス検索とは

**インデックス検索**とは、あらかじめファイル情報をデータベース（インデックス）に蓄え、**大量のファイルからでも一瞬で検索結果を返す**機能です。

| 検索モード | 特徴 | 向いている用途 |
| :--- | :--- | :--- |
| **通常検索** | その場でフォルダを走査。常に最新だが、大きいフォルダは時間がかかる | 小〜中規模のフォルダを開いた状態で検索 |
| **インデックス検索** | 事前作成のインデックスを使用。非常に高速だが更新チューニングが必要 | 登録済み複数フォルダを横断して横串検索 |

インデックス機能は **Lucene.NET**（Apache Lucene の .NET 移植版）をベースとした全文検索エンジンで、日本語ファイル名やパスも高精度に検索できるようチューニングされています。

### 4-2. 🔄 インデックスのライフサイクル

| フェーズ | 説明 |
| :--- | :--- |
| **登録** | 「インデックス検索設定」(**Ctrl+Shift+4**) でフォルダを追加。この時点ではインデックス未作成 |
| **初回インデックス** | 「未インデックスを作成」でディスク上のファイルをスキャンしインデックスを作成 |
| **差分更新** | 設定した更新モードに応じて即時検知／一定間隔／手動でインデックスを更新 |
| **フル再構築** | メニュー（⋯）→「フル再構築を実行…」で全削除→再作成。最短間隔設定以内は再スキャンスキップ |
| **一時停止・再開** | 新規作成を止めたり再開。既存インデックスは維持され、検索は停止中も可能 |

**インデックスの状態表示:**

| 状態 | 意味 |
| :--- | :--- |
| **未インデックス** | まだ一度もインデックスを作成していない。検索結果に含まれない可能性あり |
| **インデックス完了** | 現時点の内容に対するインデックス作成が完了 |
| **差分更新中** | 変更検出や一定間隔スキャンで最新状態に追従中 |
| **一時停止中** | 更新を一時停止中。既存インデックスはそのまま利用可能 |

### 4-3. 📋 インデックス検索設定ビューの詳細

インデックス検索設定ビュー (**Ctrl+Shift+4**) で、対象フォルダの登録・削除・状態確認・手動更新を行います。

- **フォルダの追加・削除**: 「フォルダを追加」でツリーから選択、「現在のフォルダを追加」で A/B ペイン表示中のフォルダを追加。一覧で選択→「削除」ボタンまたは右クリックで削除。
- **右クリックメニュー**:
  - **インデックスを再作成**: インデックスを**完全削除→再スキャン**。削除済みファイルの反映やバージョンアップ時に使用。
  - **インデックスを差分更新**: 既存維持のまま**新規・更新分のみ反映**。日常的な更新はこちらを優先。
- **インデックスの状態ブロック**: 状態（完了／差分更新中／一時停止中）と総件数を表示。検索対象 0 件時は総件数 0。「未インデックスを作成」「一時停止」「再開」で制御。メニュー（⋯）から「フル再構築を実行…」「インデックスの設定を開く」にアクセス。

### 4-4. ⚙️ アプリ設定「インデックス & 検索結果の鮮度」の詳細

アプリ設定 (**Ctrl+Shift+O**) の「3. インデックス (Indexing & Freshness)」で設定できます。

#### 4-4-1. インデックスの更新タイミングと負荷

| オプション | 向いているケース | 内部動作 |
| :--- | :--- | :--- |
| **変更を自動で取り込む** | 頻繁に更新されるディレクトリを常に最新で検索したい | FileSystemWatcher で変更を検出し**差分更新**。大量更新が続くと負荷増 |
| **一定間隔でまとめて更新する** | 夜間にまとめて変更が入る共有フォルダ等 | 1h / 2h / 4h / 1day ごとに全体スキャン。CPU・ディスク負荷は一時的 |
| **必要なときだけ更新する** | バックグラウンド処理を極力抑えたい | 自動更新なし。手動実行時のみ更新 |

#### 4-4-2. パフォーマンスと負荷

- **省エネモードでインデックスを更新する**: 一定件数ごとに待機を入れ、CPU・ディスクの連続使用を回避。
- **ネットワークドライブは軽めに処理する**: UNC / マップドドライブに長めの待機を入れ、**帯域・サーバ負荷を抑えつつ**更新。

#### 4-4-3. 検索結果の鮮度

- **できるだけ最新の状態を保つ（鮮度優先）**: ON 時、検索実行と並行してバックグラウンドで差分更新・未インデックス作成を自動実行。検索速度を維持しつつ、徐々にインデックスを最新化します。
- **インデックスが古い可能性があるときにメッセージを表示する**: ON 時、インデックス検索モード＋鮮度優先 ON＋未インデックスフォルダがある場合のみ、検索結果上部に黄色の警告バーを表示。OFF 時は常に非表示。表示された警告は **×** で一時的に非表示（そのタブ内のみ有効）。

### 4-5. 🔄 検索実行時の動き（通常検索 vs インデックス検索）

**インデックス検索タブで Enter を押したときの流れ:**

1. **既存インデックス**だけで高速にヒット候補を取得→結果一覧に表示。
2. （通常検索モードのみ）必要に応じてインデックスを作成し直し、結果が変われば一覧を差し替え。
3. 「鮮度優先」ON 時、バックグラウンドで差分更新を実行→次回以降の精度と鮮度を向上。

**通常検索**では、未インデックスの場合はまずインデックスを作成してから検索実行。インデックス済みなら即座に結果を表示します。

### 4-6. ⚠️ 検索結果の鮮度警告バー

検索結果上部の黄色のバーは**「結果が古い可能性がある」ことを示す注意喚起**です。

- 対象フォルダの一部がまだインデックスされていない場合のみ表示。
- インデックス自体は安全に利用可能。「未インデックスを作成」で一度作成しておくと今後安定します。
- 不要な場合は **×** で閉じるか、アプリ設定で「メッセージを表示する」を OFF に。

### 4-7. 🛠️ トラブルシューティングとベストプラクティス

| 現象 | 考えられる原因 | 確認・対応 |
| :--- | :--- | :--- |
| 検索してもヒットしない | インデックス未作成、または対象フォルダ未登録 | インデックス検索設定で登録→「未インデックスを作成」を実行 |
| 結果が明らかに古い | 削除・移動したファイルがインデックスに残存 | 該当フォルダを右クリック→「インデックスを再作成」 |
| 警告バーが頻繁に出る | 未インデックスのフォルダが登録されている | **×** で閉じるか、設定で OFF に。または「未インデックスを作成」で作成しておく |
| サーバーが重い／バックグラウンド負荷が心配 | 鮮度優先や自動更新が頻繁に動作 | 「必要なときだけ更新する」＋「鮮度優先」OFF。ネットワークドライブは「軽めに処理」を ON |

### 4-8. 🧩 技術的なしくみ（開発者・上級者向け）

> **注:** このセクションは技術的な詳細に興味がある方向けです。操作や設定の理解には不要なため、読み飛ばしても問題ありません。

- **技術スタック**: Lucene.NET 4.8。`IndexService` シングルトン（`App.IndexService`）で管理。`index` フォルダに `segments_N`、`_N.cfs`、`write.lock`、`zenith.db`（履歴DB）、`indexed_roots.json`（ルート永続化）で構成。
- **スレッド・例外**: `SemaphoreSlim(1,1)` で同時実行 1 件に制限。fire-and-forget タスクには `ContinueWith(OnlyOnFaulted)` で例外を観測。
- **パフォーマンス**: 省エネ／ネットワーク低優先モードでバッチサイズと待機時間を調整。`.git`、`node_modules` は自動スキップ。
- **検索精度**: `name` と `name_raw` の 2 フィールドで日本語対応。パースエラー時は AND PhraseQuery やワイルドカードにフォールバック。

### 4-9. 💾 設定バックアップ・リカバリ

アプリ設定（**Ctrl+Shift+O**）の「**4. 設定ファイルリカバリ**」で管理します。

#### バックアップの作成

| タイミング | 動作 |
| :--- | :--- |
| **アプリ終了時（自動）** | `backups/settings_yyyyMMdd_HHmmss.json` が自動生成 |
| **「今すぐバックアップ」ボタン** | 現在の `settings.json` を即時バックアップ。要約は「手動バックアップ」 |

#### 変更概要（差分サマリ）

各バックアップには、直前との差分を日本語で示す概要が `.desc` ファイルに保存されます（例:「Aペインホーム、検索挙動を更新」）。初回は「初回バックアップ」。

#### 「復元...」ダイアログ

「復元...」ボタンでバックアップ一覧ダイアログを表示。

- **列構成**: 日時 / ロック状態（錠アイコン）/ 変更概要
- **ダブルクリック**: 確認ダイアログ後、選択時点に復元。再起動を促すダイアログで「はい」→自動再起動
- **右クリック→ロック / ロック解除**: ロック済みバックアップは 30 日自動削除の対象外。重要な設定はロックして保護してください

#### 自動クリーンアップ

起動時に作成から **30 日以上**経過したバックアップを自動削除。`.lock` ファイルが存在するバックアップ（ロック済み）は削除されません。

## 5. 💼 ワーキングセット

> **A/B ペインの全状態を「プロジェクト」として保存・瞬時切り替え** — タブ・パス・ロック・表示モード・ソート・選択・スクロール位置をまるごと保存します。

ナビペインのブリーフケースアイコン（**Ctrl+Shift+5**）で表示。ビュー上部にはガイドテキストが表示されます。

### 5-1. 💾 保存

ナビペイン上部の「現在の状態を保存」ボタンをクリックし、名前を入力すると A/B ペインの全状態が保存されます。`settings.json` に永続化され、再起動後もすべて保持。

### 5-2. 👁️ プレビューと確定（ロールバック機能）

一覧からセットをクリックすると、即座に A/B ペインがその状態に切り替わります。**「確定させるまで元の状態を汚さない」**のが最大の特徴です。

- **プレビュー状態**: セットをクリックすると一時的な「プレビュー」となり、画面下部にチャコールブラック（#545B64）の専用フッターバーが表示。
- **適用**: フッターの「✔ このセットを適用する」で現在の作業状態として確定。
- **キャンセル（ロールバック）**: フッターの「✗ 元の状態に戻す」、またはナビペインで他のビューに切り替えると、**読み込み直前の状態（すべてのタブ・選択位置・スクロール状況）に自動復元**されます。

> 💡 「プロジェクト A の作業中に、プロジェクト B の資料をちょっと確認してすぐに戻る」といったコンテキストスイッチが極めてスムーズに行えます。

### 5-3. 🔧 管理

- **名前の変更**: セットを右クリック→「名前の変更」。
- **削除**: セットを右クリック→「削除」、または選択してゴミ箱アイコン。

## 6. ⌨️ キーボードとマウスの連携

> **キーボード操作とマウス操作の干渉を排除** — 意図したペインへ確実に操作でき、キーボード中はマウスホバーの誤動作を防ぎます。

- **ペイン・ナビのフォーカスはクリックで切り替え**: ナビペインや A/B ペインにフォーカスを移すには**マウスクリック**、または **Tab** / **←** / **→** キー。ホバーではフォーカスは移動しません。
- **一覧へのフォーカス**: A/B ペインにフォーカスがあるがファイル一覧にフォーカスがないとき（起動直後やタブクリック後等）、矢印キー・**Enter**・**Delete**・**F2** 等を押すと一覧にフォーカスが移り、押したキーがそのまま有効。パス入力欄や検索ボックスにフォーカス中はそれらに使われます。
- **キーボード操作中のマウス干渉排除**: 矢印・Enter・ショートカット等でキーボード操作を開始すると「キーボード操作モード」に入り、マウスカーソルが非表示に。ホバーだけでフォーカスが移動することを防ぎます。
- **マウス操作への自動復帰**: マウスを約 **3px 以上**動かすとカーソルが再表示され通常操作に復帰（わずかな振動では復帰しないデッドゾーン付き）。モーダルダイアログ表示時もカーソルは自動復元されます。
